Perfeito — vamos montar **queries de conferência** bem objetivas, do tipo que dá segurança de que a importação bate com o CSV 👍
Vou te passar em **ordem prática**, da mais geral para a mais específica.

---

## 1️⃣ Conferir quantidade total de registros importados

Isso garante que o CSV foi lido inteiro.

```sql
SELECT COUNT(*) AS total_registros
FROM ibge_dados;
```

👉 Compare com:
`nº de produtos × nº de regiões × 2 (total + familiar)`

---

## 2️⃣ Conferir por produto (com nome, não id)

Aqui você vê se os `produto_id` estão corretos.

```sql
SELECT
  p.nome AS produto,
  COUNT(*) AS registros
FROM ibge_dados i
JOIN produtos p ON p.id = i.produto_id
GROUP BY p.nome
ORDER BY p.nome;
```

🔎 Se algum produto tiver `0` ou número estranho → problema de vínculo.

---

## 3️⃣ Conferir um produto específico (ex.: Mandioca em Alegrete)

Essa é a query “cirúrgica” 👇

```sql
SELECT
  r.nome AS regiao,
  p.nome AS produto,
  i.familiar,
  i.estabelecimentos,
  i.valor_vendas
FROM ibge_dados i
JOIN produtos p ON p.id = i.produto_id
JOIN regioes r ON r.id = i.regiao_id
WHERE p.nome = 'Mandioca (aipim, macaxeira)'
  AND r.nome = 'Alegrete'
ORDER BY i.familiar;
```

Você deve ver **duas linhas**:

* `familiar = 0` → total
* `familiar = 1` → agricultura familiar

---

## 4️⃣ Conferir se existem registros “órfãos” (erro grave)

Isso detecta ids quebrados.

```sql
SELECT COUNT(*) AS registros_orfaos
FROM ibge_dados i
LEFT JOIN produtos p ON p.id = i.produto_id
WHERE p.id IS NULL;
```

✅ Resultado esperado: `0`

---

## 5️⃣ Conferir valores NULL (esperado, mas bom validar)

```sql
SELECT
  COUNT(*) AS total,
  SUM(CASE WHEN estabelecimentos IS NULL THEN 1 ELSE 0 END) AS estab_null,
  SUM(CASE WHEN valor_vendas IS NULL THEN 1 ELSE 0 END) AS valor_null
FROM ibge_dados;
```

Serve para confirmar se os `NULL` aparecem onde o CSV realmente não tinha dado.

---

## 6️⃣ Conferir distribuição por região e familiar

Essa é ótima para “bater o olho”.

```sql
SELECT
  r.nome AS regiao,
  i.familiar,
  COUNT(*) AS registros
FROM ibge_dados i
JOIN regioes r ON r.id = i.regiao_id
GROUP BY r.nome, i.familiar
ORDER BY r.nome, i.familiar;
```

Cada região deve ter:

* mesmo nº de registros para `familiar = 0` e `1`

---

## 7️⃣ Conferir soma de estabelecimentos por região (sanidade)

```sql
SELECT
  r.nome AS regiao,
  SUM(i.estabelecimentos) AS total_estabelecimentos
FROM ibge_dados i
JOIN regioes r ON r.id = i.regiao_id
WHERE i.familiar = 0
GROUP BY r.nome
ORDER BY r.nome;
```

Se algo estiver **zero ou NULL onde não deveria**, aqui aparece rápido.

---

## ✅ Checklist final de “importação OK”

A importação está correta se:

* ✔️ Não há registros órfãos
* ✔️ Cada produto tem registros nas 4 regiões × 2 tipos
* ✔️ `NULL` só aparece onde o CSV estava vazio
* ✔️ Totais ≥ familiares (nunca o contrário)
